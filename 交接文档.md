# SymbolFlow 交接文档

## 项目信息

- **仓库**: https://github.com/chleya/SymbolFlow
- **类型**: 单文件 HTML 工具（D3.js 可视化 + LLM 滚动联想）
- **主要文件**: `index.html`（唯一文件，约 1100 行）
- **依赖**: D3.js 7.x CDN，无构建步骤

打开 `index.html`（通过本地服务器，否则 localStorage 不共享）：

```bash
npx serve .
# 访问 http://localhost:3000
```

种子词输入框 → 回车 → 自动游走 → D3 力导向图渲染。

## 核心架构

### 文件结构

```
index.html
├── HTML/CSS（UI 组件）
├── D3.js force simulation（画布渲染）
├── deepHook() — LLM 钩子层
├── walk() — 主游走循环
├── 历史记录（localStorage）
└── 导出功能
```

### 数据结构

```javascript
// 节点
{ id: 'n1', label: '自由', fixed: false }

// 边
{ from: 'n1', to: 'n2', reason: '自由常与选择相连', type: '因果' }

// 步骤记录
{ step: 1, from: '自由', to: '选择', fromId: 'n1', toId: 'n2',
  reason: '自由常与选择相连', type: '因果',
  ts: '2026-04-19T...',
  diagnostics: { loops: 3, halt_reason: 'stable_candidate', drift_score: 0.12, ... } }
```

### 状态变量

| 变量 | 含义 |
|------|------|
| `Gnodes` | 当前图节点数组 |
| `Gedges` | 当前图边数组 |
| `steps` | 历史步骤记录（含 diagnostics） |
| `curN` | 当前游走位置的节点 ID |
| `selN` | 选中节点的节点 ID |
| `running` | 是否正在自动游走 |
| `nidCtr` | 节点 ID 计数器，下一个可用编号 = nidCtr + 1 |

## deepHook() 详解

### 签名

```javascript
async function deepHook({ seed, current, path, maxLoops, temperature })
```

### 内部流程（Prelude → Recurrent Loop → Coda）

```
1. Prelude: 构造重注入上下文
   - seed / path / anchor_summary / drift_patterns

2. Loop 1..maxLoops（默认4轮）:
   - 生成 3 个候选 { next, reason, type, confidence }
   - 评分：depth优先 + novelty + drift惩罚 + 等价循环惩罚
   - halting 检查：
     · equivalence_loop：连续2轮 top-1 相同 → break
     · stable_candidate：top-1 显著高于第二名 → break
   - drift_seen 标记（不 halt，继续探索）
   - 轻微升温（temperature + 0.1 * loop）

3. Coda: 从全局评分最高候选选最终结果
   - drift 命中的候选降低 score 后仍可选（不是直接排除）
   - drift_seen 影响最终 halt_reason 显示
```

### drift_patterns（漂移禁区）

```javascript
const DRIFT_PATTERNS = [
  '无名英雄', '英雄', '壮举', '勇气',
  '金钱', '欲望', '贪婪', '本能', '生存',
  '权利义务', '责任', '法律', '政治',
  '时间', '流逝', '消逝', '流势',
];
```

### 返回值

```javascript
{
  next: "清净心",
  reason: "无所住是因，清净心是果",
  type: "因果",
  diagnostics: {
    loops: 3,              // 实际循环次数
    halt_reason: 'stable_candidate', // stable_candidate | equivalence_loop | max_loops | drift_seen
    drift_score: 0.12,    // 0-1，候选命中漂移模式的程度
    candidate_count: 12,  // 总候选数（3个/轮 × 4轮）
    candidates: [...],    // 前10个候选，用于调试
  }
}
```

### halt_reason 语义

| 值 | 含义 |
|----|------|
| `stable_candidate` | 候选已收敛，提前停止 |
| `equivalence_loop` | 检测到等价循环，提前停止 |
| `drift_seen` | 见过 drift 命中，但跑到 max_loops 才停 |
| `max_loops` | 跑满所有循环 |
| `no_candidates` | API 无有效响应，fallback |

## 已修复的结构性 Bug

### 1. 节点 ID 重复（nidCtr 污染）

**问题**: `startWalk()` 创建根节点 `id:'n1'`，`nidCtr` 仍为 0，下一个节点变成 `id:'n1'`（重复）。

**修复**: `startWalk()` 和 `replay()` 末尾加 `nidCtr = 1`。

### 2. loadRec() 边重建错误

**问题**: 边用 `Gnodes[Gnodes.length-1].id` 作 from，这是新节点自身，导致自环 `nX → nX`。

**修复**: 用 `labelToId()` 映射从 steps 的 from/to 标签重建节点和边。

### 3. deepHook drift 语义不一致

**问题**: drift 命中设 `haltReason = 'drift_detected'` 但不 break，导致诊断和实际停止原因不符。

**修复**: 引入 `driftSeen` 标记，最后合并为 `halt_reason: 'drift_seen'`。

### 4. Coda 只取最后一轮

**问题**: 注释说"从所有候选选"，实际只取 `loopCandidates[last]`。

**修复**: `allScoredCandidates` 全局收集，从评分最高选。

### 5. XSS 注入面

**问题**: 历史列表渲染 seed 用 `innerHTML`，未转义。

**修复**: `escHtml()` 转义后再渲染。

## deepHook 和 hook 的关系

- `hook()` 仍存在，保留向后兼容，未被调用
- `walk()` 直接调用 `deepHook()`
- 若要回退到单次联想模式，修改 `walk()` 中的调用即可

## 滚动测试结果（已归档）

`滚动测试记录.md` 记录了 6 轮测试结果。

**最深的链**:
- 归根复命 → 落叶归根 → 周而复始 → 循环 → 周期 → 节律 → 节奏
- 一切有为法，如梦幻泡影 → 缘起性空 → 诸法空相 → 不生不灭 → 永恒（跨经联动：金刚经→心经）
- 为学日益，为道日损 → 悟道 → 明心见性

**死穴词/句**:
- 单字：空、流、觉（thinking 发散，token 耗尽）
- 长句：若夫乘天地之正...（16字，太长）、一切贤圣皆以无为法（复杂）、无智亦无得（token 耗尽）
